## 2. 수익률이 큰 종목과 작은 종목의 차이

`backtest_reason.py --input_csv backtest_type1.csv --top_n 5` 기준 관찰 결과:

### 공통 패턴

- **상위 수익률 종목**
  - 첫 진입 후 장기 보유로 이어지는 경우가 많음
  - 성과의 대부분이 **미실현손익**
  - 기간 자체의 주가 상승률이 매우 큼

  - **사고판수익률(닫힌 거래 기준)** 이 음수로 누적
  - 종료 시점이 고점 대비 큰 조정 구간인 경우가 많음

## 3. 첫 매수일에 Best/Worst가 갈리는가?

### 본 결론

**첫 매수일만으로 Best 5 / Worst 5 를 확정적으로 가르기는 어렵다.**

- 첫 매수일 거래량 급증은 상위/하위 양쪽에서 모두 나타남
- 첫 매수일 이격률도 상위/하위 양쪽에 모두 존재
- 더 중요한 것은 **첫 매수 후 추세가 얼마나 오래 유지되는가** 이다

## 4. + 구간 / - 구간 길이와 성과 차이

`Close > MA10M` 을 `+ 구간`, `Close <= MA10M` 을 `- 구간` 으로 보고 각 종목의 연속 구간 길이를 계산했다.

### Best 5 vs Worst 5 평균

| 구분 | 평균 수익률 | 평균 +구간 길이 | 평균 최대 +구간 | 평균 -구간 길이 | 연간 전환 횟수 | +구간 비중 |
|---|---:|---:|---:|---:|---:|---:|
| Best 5 | 363.82% | 192.85일 | 303.6일 | 90.77일 | 1.48회 | 73.84% |
| Worst 5 | -9.49% | 135.80일 | 299.2일 | 67.40일 | 2.39회 | 60.44% |

### 해석

- **평균 +구간 길이**는 수익률과 양의 상관 (`corr ≈ +0.38`)
- **연간 전환 횟수**는 수익률과 음의 상관 (`corr ≈ -0.28`)
- `- 구간 길이` 자체는 설명력이 약함
- 핵심은 **한 번 위로 올라간 뒤 오래 버티는가**, 그리고 **왕복이 적은가** 이다

### 구간 길이 기준 분기점

- 평균 `+ 구간`이 **40거래일 전후**부터 성과 차이가 나타나기 시작
- 평균 `+ 구간`이 **80거래일 이상**이면 성과가 더 확실히 벌어짐
- 연간 전환 횟수가 **11~15회 이상**이면 빈번 반복 구간으로 들어가며 성과가 약해지는 경향

실전 해석:

1. 첫 매수 후 `+` 상태가 **약 40거래일 이상 유지**되면 장기형으로 갈 가능성이 커짐  
2. `+` 상태가 **80거래일 이상 유지**되면 강한 승자 구간으로 갈 가능성이 더 커짐  
3. 반대로 전환이 잦아 연간 **11~15회 이상** 뒤집히면 성과가 약해질 가능성이 큼

## 8. `+`가 매우 길고 이격률도 매우 큰 종목은 너무 많이 오른 것인가?

### 짧은 의견

**네, 신규 진입 관점에서는 “많이 오른 상태”로 보는 게 맞다.**  
다만 **기존 보유 관점**에서는 무조건 과열이라고 단정하기는 어렵다.

이 전략에서는:

- `+ 구간이 길다` = 추세가 강하게 지속되고 있다는 의미
- `이격률이 크다` = 현재 가격이 MA10M 대비 많이 앞서 있다는 의미

따라서 이 둘이 동시에 크면:

1. **기존 보유자 입장**
   - 강한 추세를 타고 있는 승자일 가능성이 큼
   - 실제로 `장기 + 고이격` 종목군은 평균수익률이 높았음

2. **신규 진입자 입장**
   - 평균회귀(되돌림) 위험이 커짐
   - MA10M과 거리가 너무 멀어 손절/재진입 판단이 어려워짐
   - 기대수익 대비 리스크가 나빠질 수 있음

### 실제 숫자 비교

연속 `+` 일수와 현재 이격률이 모두 상위 20%인 종목 17개를 따로 보면:

| 집합 | 종목수 | 평균연속일수 | 평균현재이격률 | 중앙수익률 | 평균수익률 |
|---|---:|---:|---:|---:|---:|
| 장기 + 고이격 | 17 | 267.41 | 74.44 | 72.99 | 142.38 |
| 나머지 | 295 | 159.01 | 28.05 | 5.30 | 15.58 |

즉,

- **이미 보유 중이면 매우 강한 승자 집단**일 가능성이 높다
- 하지만 **지금 새로 사기에는 과열/되돌림 위험이 큰 집단**으로 봐야 한다

### 실전 해석

- **보유 관점:** 추세 훼손 전까지 보유 검토 가능
- **신규 진입 관점:**  
  - 너무 긴 `+` 구간 + 높은 이격률은 추격매수보다  
  - 조정 후 재돌파, 또는 이격률 완화 뒤 재확인이 더 유리할 가능성이 높다

결론적으로:

> **“길고 높은 + 구간”은 강한 승자라는 뜻이기도 하고, 동시에 신규 진입자에게는 과열 구간일 수 있다.**  
> 따라서 같은 숫자라도 **보유자**와 **신규 진입자**의 해석은 달라야 한다.

현재 `+ 40일 이상` 목록 재생성:

```bash
uv run python - <<'PY'
from analyze import load_kospi_list, load_sp500_list, ETF_SYMBOLS, STOCKS_DIR, US_STOCKS_DIR
from backtest_type1 import load_price_frame
import pandas as pd

items = []
for _, row in load_kospi_list().head(200).iterrows():
    items.append(('KOSPI 200', str(row['Code']), str(row['Name']), STOCKS_DIR / f"{str(row['Code'])}.csv"))
for _, row in load_sp500_list().iterrows():
    items.append(('S&P500', str(row['Symbol']), str(row['Name']), US_STOCKS_DIR / f"{str(row['Symbol'])}.csv"))
for sym in ETF_SYMBOLS:
    items.append(('ETF', sym, sym, US_STOCKS_DIR / f'{sym}.csv'))

rows = []
for group, ticker, name, path in items:
    df = load_price_frame(path)
    if df is None or df.empty:
        continue
    state = (df['Close'] > df['MA10M']).astype(int)
    if int(state.iloc[-1]) != 1:
        continue
    streak = 1
    for v in reversed(state.iloc[:-1].tolist()):
        if int(v) == 1:
            streak += 1
        else:
            break
    if streak >= 40:
        close = float(df['Close'].iloc[-1])
        ma = float(df['MA10M'].iloc[-1])
        rows.append({
            '그룹': group,
            '티커': ticker,
            '종목명': name,
            '연속+일수': streak,
            '기준일': df.index[-1].strftime('%Y-%m-%d'),
            '현재가': round(close, 2),
            '10월이평': round(ma, 2),
            '현재이격률(%)': round((close - ma) / ma * 100, 2),
        })

pd.DataFrame(rows).sort_values(['그룹', '연속+일수'], ascending=[True, False]).to_csv(
    'data/current_plus_40.csv', index=False, encoding='utf-8-sig'
)
PY
```

## 10. Type2 파라미터 탐색: 총수익 최적과 실전형의 차이

`2024-01-01 ~ 2026-04-11` 구간에서 `plus_days`, `minus_days` 를 `1~40`까지 전수 탐색했다.

### 요약

| 구분 | plus_days | minus_days | 총수익률 | 사고판수익률 | 해석 |
|---|---:|---:|---:|---:|---|
| 총수익률 최대 | 23 | 40 | 66.97% | -15.29% | 매도를 매우 늦춰 장기 보유 이익을 극대화 |
| 실전형 균형안 | 33 | 5 | 40.47% | 0.40% | 거래 횟수를 크게 줄이면서 닫힌 거래 성과도 방어 |
| 기본값(type1 동일) | 1 | 1 | 8.86% | -0.74% | 너무 자주 반응해 왕복매매가 많음 |

### 해석

- `minus_days` 를 길게 잡을수록 총수익률은 크게 좋아졌지만, 최고값이 `40일` 경계에서 나와 **더 길게 잡으면 총수익률이 더 올라갈 가능성도 남아 있다**
- 대신 닫힌 거래 기준 성과(`사고판수익률`)는 악화되었다
- 즉, `23/40` 은 **백테스트 총수익 극대화형**이고, `33/5` 는 **실제 운용에서 과도한 지연과 과열 추격을 줄이는 균형형**으로 보는 것이 맞다

## 11. 실전형 `33/5` 기준 종목 선정과 운용 방법

이번 목적은 **시장별 10개 이하**로 좁혀서, 교체를 자주 하지 않으면서도 성과를 내는 방식이다.

### 먼저 확인된 한계

- 현재 저장된 종목 CSV는 `Close`, `MA10M` 위주이며 **과거 거래량 이력이 저장되어 있지 않다**
- 그래서 이번 정리는 **추세 지속성, 전환 빈도, 백테스트 수익률, KOSPI의 시가총액/배당률** 중심으로 했다
- KOSPI는 `FinanceDataReader`와 `NAVER/FINSTATE`로 현재 시총/배당률을 붙였고, 미국/ETF는 로컬 데이터 한계로 배당률/시총을 정량 반영하지 못했다

### 선별 규칙

#### Core 후보 (지금 실제 보유 시작 후보)

- 현재 `+` 상태
- 현재 `+` 연속일수 `20일 이상`
- type2 `33/5` 수익률 양수
- `buy_count <= 10`
- `switches_per_year <= 4`

#### Watchlist 후보 (성과는 좋지만 전환이 다소 잦음)

- 현재 `+` 상태
- 현재 `+` 연속일수 `20일 이상`
- type2 `33/5` 수익률 양수
- 위의 turnover/switch 조건 일부 미달

### KOSPI 200 Core 5

## 12. 실제 운용 추천

### 지금처럼 10개 이하로 운용할 때

1. **신규 편입은 Core 후보만**
2. **기존 보유는 `-` 상태가 5일 연속될 때만 매도**
3. **주간 1회만 교체 판단** — 매일 바꾸지 않음
4. **한 번의 점검에서 최대 2종목만 교체**
5. **새 후보가 기존 보유보다 명확히 좋을 때만 교체**

### 더 좋은 넣고/빼기 방법

- **매수 조건**
  - `+`가 33일 연속된 종목만 신규 편입 검토
  - 단, 현재이격률이 너무 큰 종목(예: `45%+`)은 신규 편입보다 watchlist에 두고 조정 후 재확인

- **보유 유지 조건**
  - 이미 들고 있는 종목은 `+`가 길고 이격률이 커도 바로 팔지 않음
  - `-` 5일 연속 또는 상대 점수 급락 시에만 교체 검토

- **교체 조건**
  - 신규 후보가
    - 현재 `+` 상태,
    - `+` 연속 33일 이상,
    - `33/5` 수익률 양수,
    - 전환 빈도/매수 횟수가 더 낮고,
    - 과열도(이격률)가 더 낮을 때만
  - 기존 보유 1종목과 교체

### 최종 의견

- **총수익 최대화 목적**이면 `23/40`
- **실제 제한된 종목 수로 운용**하려면 `33/5`가 더 낫다
- 특히 이번 데이터에서는 **“많이 사고파는 것”보다 “좋은 추세를 오래 보유하는 것”** 이 더 중요했다
- 따라서 실제 포트폴리오는 **KOSPI core 5 + S&P500 core 4 + ETF watchlist 1** 정도로 시작하는 구성이 가장 자연스럽다

## 15. type4: 시가총액 상위 조건을 붙인 전략

`backtest_type4.py` 를 추가했다.

- **KOSPI:** 시가총액 상위 `30`
- **S&P500:** 시가총액 상위 `100`
- **매수:** 기간 안에서 `- -> +` 전환이 발생했고 상위 조건도 만족할 때
- **매도:** `+ -> -` 전환 시

시작 시점에 이미 `+` 인 종목은 **매수하지 않는다**. 이 규칙으로 삼성전자처럼 사실상 항상 top 30 안에 드는 종목도 type1과 동일한 매매 횟수로 맞춰진다.

### 시가총액 계산 방식

과거 시점의 정확한 시가총액 데이터가 없으므로 아래 방식으로 근사했다.

1. 현재 시가총액 ÷ 현재가 = 유통주식수 근사
2. 그 유통주식수 × 과거 종가 = 과거 시점 근사 시가총액
3. 이 값으로 날짜별 상위 30 / 상위 100 순위 계산

### 현재 구현상의 자금 기준 주의점

`backtest_compare.py` 에서 **type4는 별도 slot 자금 기준**을 사용한다.

- KOSPI: 종목당 `1,000만원 / 30`
- S&P500: 종목당 `10,000달러 / 100`

즉, type4는 다른 전략과 달리 **상위 후보 최대 보유 수에 맞춘 분할 자금**으로 계산된다.
